Proof of Consent (Opt-In) Documentation

Last updated: March 7, 2026

Consent Method

This is a private, family-use application with only two recipients. Both recipients are the sole owners and operators of the application. Consent is provided directly by each recipient through the act of manually configuring their own phone number in the application's environment configuration file.

How Consent Is Given

Each recipient explicitly adds their own phone number to the application's .env configuration file. This is a deliberate, manual action that requires direct access to the application's server. No phone numbers can be added without physical or SSH access to the device running the application.

The configuration file contains clearly labeled fields:

RYAN_PHONE=+1XXXXXXXXXX    # Parent 1 cell number
NIKKI_PHONE=+1XXXXXXXXXX   # Parent 2 cell number

By entering a phone number into these fields, the recipient explicitly consents to receive automated SMS notifications about calendar updates.

How to Revoke Consent (Opt-Out)

  • Text STOP to the sending number at any time to immediately stop all messages.
  • Remove the phone number from the .env configuration file.
  • Text HELP for assistance.

Scope of Messaging

  • Recipients: Only 2 family members (the application owners)
  • Message type: Automated calendar update notifications only
  • Frequency: 0-5 messages per day, batched every 30 minutes
  • No marketing or promotional messages are ever sent
  • No third-party phone numbers are ever added
